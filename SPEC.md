# SPEC.md — GODA BRIDGE TikTok予約専用LP

> **役割**: このファイルはAIと人間の「共有言語」として機能する。あいまいな表現を使わず、実装者（AI）がそのままHTMLを書ける粒度で記述する。

---

## 1. Design Direction

### コンセプト

**「スリルの予告」**

渡る前から体験が始まっている感覚を与えるLP。施設を紹介するのではなく、「自分がそこにいる瞬間」を想像させる。TikTokで動画を見て高まった気持ちを、LPがそのまま受け止めて予約へ変換する。

---

### トーン（対比形式）

| ではなく | こう見せる |
|---------|-----------|
| 施設案内ページ | 体験の入口 |
| 安心感・安全訴求 | スリルの期待感 |
| 情報量の多い説明 | 感じさせる映像と余白 |
| 落ち着いた観光施設 | 非日常・高揚感 |
| 「行けるかな」という不安 | 「行かないと損だ」という衝動 |

---

### フォント

#### 欧文（Google Fonts）

| 用途 | フォント名 | ウェイト | 例 |
|------|-----------|---------|-----|
| 大見出し・数字 | `Bebas Neue` | 400（=Boldに相当） | 860万 / 55M / GODA BRIDGE |
| サブ見出し・ラベル | `Oswald` | 700 | EXPERIENCE / BOOK NOW |

```html
<link href="https://fonts.googleapis.com/css2?family=Bebas+Neue&family=Oswald:wght@400;700&display=swap" rel="stylesheet">
```

#### 和文（Google Fonts）

| 用途 | フォント名 | ウェイト | 例 |
|------|-----------|---------|-----|
| 和文見出し | `Noto Sans JP` | 900 | 渡るたびに、また来たくなる。 |
| 和文本文 | `Noto Sans JP` | 400 | 所要時間・注意事項・説明文 |

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;900&display=swap" rel="stylesheet">
```

#### 使い分けルール
- **数字・英語キーワード**: Bebas Neue — 大きく・太く・インパクト重視
- **和文キャッチコピー（H1・H2）**: Noto Sans JP 900
- **ボタンテキスト**: Oswald 700（英語）または Noto Sans JP 700（日本語）
- **説明文・注釈**: Noto Sans JP 400
- フォントサイズは `clamp()` で流動的に設定（例: `clamp(2rem, 8vw, 6rem)`）

#### テキスト内容

#### インタラクション仕様

- **動画**: `<video autoplay muted loop playsinline>` — スマホでも自動再生されるよう `playsinline` 必須
- **動画ソース**: MP4（吊り橋を横から見る全景、または橋の上を歩く視点映像）※素材差し替え前提でプレースホルダー
- **オーバーレイ**: `rgba(10,10,10,0.45)` を動画上に被せてテキスト可読性を確保
- **H1アニメ**: ページロード後 0.3s 遅延で `opacity: 0 → 1` + `transform: translateY(20px) → translateY(0)` （`transition: 0.8s ease`）
- **CTAボタン**: `background: #FF5500`、ホバー時 `background: #FF7733` + `transform: translateY(-2px)` + `box-shadow: 0 8px 30px rgba(255,85,0,0.4)`
- **SCROLL誘導**: 上下にゆっくり動くバウンスアニメ（CSS `@keyframes`）、スクロール開始後フェードアウト

#### 実装上の注意点

- 高さ: `height: 100svh`（iOSのアドレスバー対応）
- 動画がない場合のフォールバック: `background: linear-gradient(180deg, #0A0A0A 0%, #1A1A1A 100%)` + 静止画
- 動画ファイルはウェブ最適化（H.264、10MB以内）

#### テキスト内容

| 要素 | テキスト |
|------|---------|
| セクションラベル | TikTokから届いた、その先へ |
| 数字1 | 860万 |
| ラベル1 | 再生されたカップル動画 |
| 数字2 | 59,300 |
| ラベル2 | TikTokフォロワー |
| 数字3 | 55 |
| ラベル3 | mの高さから見る絶景 |
| 補足コピー | この数字の先に、あなたの体験がある。 |

#### インタラクション仕様

- **カウントアップアニメ**: セクションがビューポートに入った時点（Intersection Observer）でカウント開始
  - 860万: 0 → 8,600,000（カンマ区切り表示） 1.5秒
  - 59,300: 0 → 59,300 1.2秒
  - 55: 0 → 55 0.8秒
- **数字の色**: `#D4A843`（ゴールド）
- **フェードイン**: 左・中央・右の数字が 0.1s ずつ遅延してフェードアップ

#### 実装上の注意点

- スマホでは3カラム → 縦積みではなく **横スクロール（`overflow-x: auto`）** または2行（860万 / 59,300 ｜ 55m の配置）
- カウントアップはJavaScriptで実装（`requestAnimationFrame`）

- 体験カードは最低 `280px` 幅を確保

## 3. グローバルなインタラクション仕様

### スクロールアニメーション

```javascript
// Intersection Observer 設定
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('is-visible');
    }
  });
}, { threshold: 0.15 });

document.querySelectorAll('.fade-up').forEach(el => observer.observe(el));
```

```css
/* アニメーション定義 */
.fade-up {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity 0.7s ease, transform 0.7s ease;
}
.fade-up.is-visible {
  opacity: 1;
  transform: translateY(0);
}
/* 遅延クラス */
.delay-1 { transition-delay: 0.1s; }
.delay-2 { transition-delay: 0.2s; }
.delay-3 { transition-delay: 0.3s; }
```

### 固定CTAバー（スクロール後に常時表示）

```
┌────────────────────────────────┐ ← 画面下部に固定
│  今すぐ予約する →              │
```

- **表示条件**: ページスクロール量が `100px` を超えた時点で `opacity: 0 → 1`、`translateY(100px) → 0`（0.3s ease）
- **非表示条件**: Section 07（#booking）がビューポートに入ったら非表示（`opacity: 0`）
- **スタイル**: `position: fixed; bottom: 0; left: 0; right: 0; z-index: 999; background: #FF5500; padding: 1rem`
- **テキスト**: `今すぐ予約する →`（Oswald 700 または Noto Sans JP 700、`color: #FFFFFF`）

### ページ遷移

- **方針**: シングルページ（SPA不使用）。セクション間移動はスムーズスクロール
- **スムーズスクロール**: `html { scroll-behavior: smooth; }`
- **アンカーリンク**: ナビゲーション不要（TikTokから直接流入するため上部ナビを省略）

---

### レスポンシブ方針

| ブレイクポイント | 幅 | 対象デバイス | 方針 |
|----------------|-----|------------|------|
| モバイル（基準） | 375px〜 | スマホ（TikTok流入メイン） | 縦スタック・タッチ操作最適化 |
| タブレット | 768px〜 | iPad | 2カラム対応 |
| デスクトップ | 1024px〜 | PC | 最大幅 `1200px`・中央寄せ |

**基本方針**:
- `min-width` ではなく `max-width` の mobile-first で記述
- タッチ操作: カルーセルは `touch-action: pan-y` でスワイプ有効化
- フォントサイズ: `clamp(最小値, vw値, 最大値)` を使用
- ボタンの最小タップ領域: `min-height: 48px`（Googleガイドライン準拠）
- 動画: スマホでは `playsinline` 必須（iOS Safari 対応）
